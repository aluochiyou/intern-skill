# Evaluation Rubric

Use this rubric to evaluate a `project-intern` session. Score each item from 1 to 5.

## Dimensions

1. **Project positioning**: Did the answer establish purpose, users, inputs, outputs, and stack?
2. **Main path focus**: Did it prioritize the core execution path before side details?
3. **Evidence quality**: Were claims tied to code, docs, commands, web sources, or clearly marked inference?
4. **Beginner support**: If needed, did it explain prerequisite terms before professional analysis?
5. **Scope control**: Did it stay on one small learning goal or a clear staged path?
6. **Engineering value**: Did it explain design tradeoffs, reliability, observability, extensibility, or evaluation value?
7. **Mode discipline**: Did it avoid opening career, audit, publishing, or frontier-transfer sections unless the user asked for them or the task clearly required them?
8. **Safety**: Did it avoid destructive actions, secret exposure, unsafe publishing, and unconfirmed deployments?
9. **Actionability**: Did it end with 1-3 concrete next steps?
10. **Skill fitness**: Did the current skill instructions and scripts make the task easier instead of adding noise?

## Interpretation

- 5: Strong and reusable.
- 4: Good with minor gaps.
- 3: Adequate but needs refinement.
- 2: Weak or inconsistent.
- 1: Failed or unsafe.

## Improvement Trigger

Propose a skill improvement only when:

- A dimension scores 3 or lower.
- The same failure appears in multiple sessions.
- A repeated action should become a deterministic script.
- The main SKILL.md is carrying details that belong in references or scripts.
- The skill repeatedly opens optional modes without enough user intent.

Prefer small changes:

1. Adjust a reference checklist.
2. Add one script option.
3. Tighten a workflow rule.
4. Add a template.
5. Only then modify the main SKILL.md.
